1. Introduction

Dossier propose depuis sa version 4 des Web Services de type soap permettant à d’autres applications ou à des intégrateurs de s’interfacer avec Dossier.

La liste des Web Services disponibles est accessible à l’adresse

http(s)://<hostname>:<port>/<context>/services

La plupart des Web Services sont également accessibles en API REST.
Si disponibles, il est conseillé d’utiliser ces APIs

2. ExtendSearch

Ce Web Service n’est disponible qu’à partir de la version 7.2 de Dossier

La wsdl est accessible à l’adresse suivante:

http(s)://<hostname>:<port>/<context>/services/extendSearch?wsdl

L’encodage des requêtes et des réponses est UTF-8.

2.1. Client

La classe com.digitech.dossier.ws.IExtendSearchWSClient de la librairie com.digitech.airs:dossier-ws-stubs:1.1.0 abstrait l’utilisation de cette API.

Comme indiqué précédemment, la wsdl est accesible.

  • La plupart des trchnologies orienté-objet (JAVA, C#, …​) supporte la génération de l’ensemble des structures des requêtes / réponses en se basant sur ce fichier. La génération de ces "stubs")_ facilite votre dévelopement mais reste optionelle.

  • Cette documentation est également optionnelle car le fichier WSDL, par définition, suffit pour connaître la structure d’entrée / sortie exacte d’un Web Service.

2.2. Authentification

Ce Web Service requiert une authentification basique.

2.3. Méthodes

search4FieldsAndAttachments

Cette méthode permet à partir d’une requête AIRS de récupérer

  • des documents

  • leurs metadata (champs)

  • et les informations sur les pièces jointes de chaque document

Listing 1. exemple de requête soap
<soapenv:Envelope xmlns:soapenv="http://schemas.xmlsoap.org/soap/envelope/" xmlns:web="http://webservices.dossier.digitech.com">
   <soapenv:Header/>
   <soapenv:Body>
      <web:search4FieldsAndAttachments>
         <web:airsRequest>D_CREAT >= 28/03/2020</web:airsRequest>
         <!--1 or more repetitions:-->
         <web:ctyCodes>CR</web:ctyCodes>
         <web:ctyCodes>PRO_CLIENT</web:ctyCodes>
         <!--Optional:-->
         <web:flowCode></web:flowCode>
         <!--Optional:-->
         <web:continuationToken></web:continuationToken>
      </web:search4FieldsAndAttachments>
   </soapenv:Body>
</soapenv:Envelope>

La réponse est retournée sous forme de chaîne de caractères.
