== {soap}

[TIP]
====
Préférez les {webservices} de type {rest} !

En effet, tout en offrant _a minima_ les mêmes services, ils ont de nombreux avantages :

* Plus faciles à utiliser.
* Moins consommateur, tant au niveau _mémoire_ que _cpu_, côté client et serveur.
* Plus efficient, rapide.
====

[TIP]
====
Comme indiqué précédemment, la *wsdl* est accessible. +

* La plupart des technologies orienté-objet ({java}, C#, ...) supporte  la génération de l'ensemble des structures des requêtes / réponses en se basant sur ce
fichier. La génération de ces _"stubs"_) facilite votre dévelopement mais reste optionelle.
* Cette documentation est également optionnelle, car le fichier _WSDL_, par définition, suffit pour connaître la structure d'entrée / sortie exacte d'un
{webservice}.
====
